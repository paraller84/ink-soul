# Wellness Routine Design — 健康作息方案设计

## 核心哲学

> **健康作息不是一张完美的时间表，而是一套在真实约束下可执行的妥协方案。**
> 目标不是"理想作息"，而是"比今天好10%的明天"。

## 第一步：约束收集（先问齐，再设计）

在设计任何作息方案前，必须收集以下全量约束。**遗漏任一类别都会导致方案被推翻重来。**

### ⚕️ 健康状况（来自体检报告）

| 指标 | 对作息的影响 | 设计含义 |
|------|------------|---------|
| 空腹血糖↑ / 糖化↑ | 胰岛素抵抗→餐后血糖峰值 | 餐后运动（散步15min）= 硬要求 |
| ALT↑ + 脂肪肝 | 肝脏代谢窗口在早晨 | 早餐不可跳过，晚餐要早 |
| 胆固醇↑ + LDL↑ | 需要规律有氧燃脂（30min+） | 每周有氧≥150min |
| 同型半胱氨酸↑ | 神经递质平衡受损→浅睡 | 补充B族→4-6周改善睡眠基础 |
| 胆囊胆固醇结晶 | 胆汁淤积→不吃早餐加重 | 🚨 早餐=非谈判项 |

### 🏢 工作约束

- **工作地点**：居家 / 办公室 / 混合 → 影响通勤时间和可插入的碎片运动
- **通勤方式**：地铁/开车/骑行/步行 → 通勤本身就是运动窗口
- **上下班时间**：精确到分钟
- **午休习惯**：✅ 保留 或 ❌ 不午休 → **不可擅自删除**用户的既有习惯
- **黄金时间**：用户的认知高峰段 → 保留为深度工作块

### 👨‍👩‍👦 家庭约束

- **子女**：接送时间、陪学习时段、学校特殊安排（如周一/三早放学）
- **配偶**：下班时间、到家时间、夫妻交流时间（👈 最容易被忽略的非谈判项）
- **阿姨/保姆**：服务时间范围（如直到20:00）→ 影响晚餐和家务安排
- **家庭晚餐**：一起吃的频率、时间点

### 🛠️ 现有固定活动（最容易被遗漏）

- **与Agent的协作时间**：用户可能每晚有固定时段优化系统
- **已有运动/爱好**：哪怕只是偶尔骑行、跑步
- **已有健康习惯**：午休、喝水习惯、特定饮食模式

## 第二步：非谈判项排序

当所有约束冲突时，按此优先级削减：

```
P0 ── 家庭关系（夫妻交流时间）
P1 ── 睡眠时长（代谢修复窗口）
P2 ── 运动（可嵌入碎片，不能消失）
P3 ── 协作时间/个人项目
P4 ── 理想中的"完美作息"细节
```

**关键决策模式**：
- 理想作息 vs 家庭现实 → 家庭优先
- 推迟睡觉 vs 削减运动 → 运动嵌入，不削减
- 单独运动时间 vs 碎片运动 → 碎片嵌入优于单独块

---

## 第三步：运动嵌入法（核心技巧）

**不为运动开辟单独时间块，而是改造已有时间缝隙。**

| 时间缝隙 | 改造方案 | 运动效益 | 时间成本 |
|---------|---------|---------|---------|
| 🌅 起床后 | 早起15min → 深蹲+俯卧撑+平板 | 力量训练 | **+15min**（代价最大，但效益最高） |
| 🚇 地铁通勤 | 走楼梯不坐扶梯 | 日常有氧 | 0（同一时间） |
| 🍱 午休前 | 饭后快走20min（而不是散步） | **中等强度有氧** | 0（替换原有散步） |
| 🛵 送娃后 | 绕小区走10min再去地铁站 | 低强度有氧+日晒 | **+10min** |
| 🚏 通勤路上 | 提前一站下车走去公司 | 中等有氧 | **+10-15min** |
| 🌞 周末 | 家庭骑行/徒步（带家人） | 主力运动 | 0（用亲子时间替代） |

**每周运动量计算模板**：
```
晨间力量：5天 × 15min = 75min
午间快走：5天 × 20min = 100min
地铁楼梯：10次 × 3min = 30min
周末主力：1天 × 30min = 30min
─────────────────────
合计：235min/周  ✅ 远超WHO 150min 推荐
```

## 第四步：针对浅睡/中途醒的设计模式

### 根因分析（基于代谢数据）

```
同型半胱氨酸↑ → 神经递质合成受阻 → 浅睡
糖化↑ → 夜间血糖波动 → 交感神经异常激活 → 凌晨醒
脂肪肝 → 肝脏解毒能力↓ → 夜间无法完成代谢修复 → 醒后疲惫
```

### 针对性的作息调整

| 措施 | 针对 | 时间线 |
|------|------|-------|
| 🍽️ 晚餐提前（睡前4h完成） | 夜间血糖稳定→深睡 | 当天 |
| 🦶 睡前泡脚15min（40-42℃） | 体温波动诱导深睡 | 3天 |
| ☀️ 晨间光照3min（醒后立即） | 重置昼夜节律 | 每天 |
| 💊 补充叶酸+甲钴胺（遵医嘱） | 降低同型半胱氨酸 | 4-6周 |

### 醒后疲惫应对

不要按贪睡闹钟——"再眯5分钟"让皮质醇混乱，醒来更累。替代方案：
1. 醒即起（闹钟放远处）
2. 晨间光照3-5min（视交叉上核重置）
3. 冷水脸15秒（三叉神经激活）
4. 温水200ml（纠正夜间脱水）

## 第五步：晚间时间预算规则

```
┌────────────────────────────────────────────┐
│  🏠 到家 ─────────────────────────── T0    │
│  ↓                                            │
│  👦 陪子女学习 ────────────────── 30-45min  │
│  ↓                                            │
│  ❤️ 夫妻交流/一起吃饭 ────────── 30-60min  │
│  ↓                                            │
│  🤖 个人项目/Agent协作 ────────── 30-60min  │
│  ↓                                            │
│  🦶 泡脚+洗漱 ─────────────────── 15-20min  │
│  ↓                                            │
│  🌙 熄灯 ─────────────────────────── T1    │
└────────────────────────────────────────────┘
```

**关键原则**：
- T1（熄灯）由家庭现实决定，不是理想值
- 如果T1比理想晚，**不恐慌**——稳定比早更重要
- 泡脚不能跳过（对浅睡效果最直接、投入最小）
- 夫妻交流优先于Agent协作（P0 vs P3）

## 第六步（NEW）：分日型设计模式

一周中每天的约束不同 → 设计多套晚间方案。

### 识别分日的依据

| 信号 | 含义 | 典型场景 |
|------|------|---------|
| 子女学校安排 | 某几天作业少→可安排额外活动 | 周一/三早放学→亲子运动日 |
| 配偶下班时间 | 某几天特殊安排→调整夫妻时间 | 固定某天社交日→需配偶早回 |
| 个人固定活动 | 每/隔周的固定社交/娱乐 | 周五社交夜 |
| 通勤差异 | 不加班日 vs 加班日 | 18:15到 vs 20:00到 |

### 分日型模板

| 星期 | 运动 | 晚间主活动 | 不同点 |
|:----:|------|-----------|--------|
| 周一 | 晨间+午间+亲子慢跑30min | 陪儿子+夫妻交流 | 🌟 亲子运动夜 |
| 周二 | 晨间+午间 | 陪学习+Agent协作+夫妻交流 | 标准日 |
| 周三 | 晨间+午间+亲子慢跑30min | 陪儿子+夫妻交流 | 🌟 亲子运动夜 |
| 周四 | 晨间+午间 | 陪学习+Agent协作+夫妻交流 | 标准日 |
| 周五 | 晨间+午间 | 弹性（社交/休息） | 可安排社交夜 |
| 周六 | 周末主力运动（骑行/慢跑30-45min） | 家庭日 | 可带家人 |
| 周日 | 积极休息（散步15min） | 家庭日+准备周一 | 低强度恢复日 |

**社交夜机制**（重要！）：
- 不需要每周，但需要固定锚点（如每两周一次周五晚）
- 固定后用户可与配偶协调：社交夜配偶早回家接替陪孩子
- 避免临时协调产生的家庭摩擦

## 第七步（NEW）：执行系统搭建（反馈即打卡）

设计完作息方案后，必须将其转化为**可执行的提醒+可追踪的反馈系统**。

### 核心机制：每条提醒即打卡点

```
各时间点自动推送提醒
  ↓ 用户回复 ✅ + 时间
记录为已执行（含时间/备注）
  ↓ 用户不回复
记录为未执行 ❌
  ↓ 周度回顾
生成执行率报告 + 调整建议
```

### 提醒内容结构

每条提醒应包含：
1. **当前时间点**（如 6:00 起床+晨练）
2. **具体动作清单**（用 □ 可勾选形式）
3. **确认提示**（"做到了回我 ✅ + 时间"）
4. **逾期含义**（"没回=没做 ❌"）

### 提醒丰富化（NEW）：知识小贴士

每条提醒后追加一条与当前活动相关的**小知识**，增加生活性和用户粘性：

**设计原则：**
- **上下文相关**：知识与提醒内容直接关联（泡脚提醒配睡眠科学，熄灯提醒配生物钟）
- **科学可信**：基于研究数据或公认生理规律（心率区间、褪黑素机制）
- **简短可消化**：1行以内，不喧宾夺主
- **轮换不重复**：每类提醒备3-4条知识，每日随机抽取

**实现模式：**
- 在提醒脚本中维护 `KNOWLEDGE_TIPS` 字典，key=提醒类型，value=知识列表
- 用 `random.choice()` 从对应列表选取
- 分日型提醒（如 Mon/Wed 亲子跑 vs 其他日）使用不同 key 获取不同领域知识

**知识领域分布（参考）：**

| 提醒 | 知识领域 | 条数 |
|:----:|---------|:----:|
| 🌅 起床 | 生理节律、空腹运动 | 3-4 |
| 🍱 午餐 | 饮食科学、血糖管理 | 3-4 |
| 🚶 快走 | 运动生理、饭后禁忌 | 3-4 |
| 🏠 到家(亲子跑) | 亲子教育、教育心理学 | 3 |
| 🏠 到家(普通) | 晚餐习惯、家庭关系 | 3 |
| 👟 慢跑 | 跑步技术、损伤预防 | 3-4 |
| 💑 交流 | 亲密关系、沟通心理学 | 3 |
| 🦶 泡脚 | 睡眠科学、体温调节 | 3-4 |
| 💤 熄灯 | 睡眠周期、生物钟 | 3-4 |

**注意事项：**
- 知识不包含价值判断或说教语气（避免"你应该""一定要"）
- 用 💡 作为前缀标识，与提醒正文视觉区分
- 用随机抽取避免"又到这条"的重复感

### 推荐的技术实现

| 组件 | 实现方案 | 说明 |
|------|---------|------|
| 提醒脚本 | no_agent=True 的 Python 脚本 | 根据当前时间+星期几输出消息文本 |
| 定时触发 | cron job（每时间点一条） | 精确到分钟 |
| 目标送达 | Feishu 群聊 | 推送到健康管理群 |
| 打卡记录 | JSON 日志文件 | `~/.hermes/cron/health-tracker/YYYY-MM-DD.json` |
| 周度汇总 | Python 脚本生成执行率 | 按任务分类统计 >= 周度回顾时推送 |

### 脚本模式

**health-reminder.py**：根据 time_key + weekday 输出对应提醒文本
- time_key = "0600"|"1200"|"1230"|"1700"|"1945"|"1955"|"2100"|"2230"|"2300"|"weekly_review"
- 1945 自动区分 Mon/Wed（亲子跑步夜）和其他日
- 1955 仅 Mon/Wed 推送

**health-tracker.py**：记录和查询执行情况
- `record <task_key> --time HH:MM --note '说明'` → 记录一次执行
- `today` → 显示当日执行概览
- `week` → 生成周度执行率报告

### 午餐互动模式

如果用户有午餐时间快走15-20min的计划：
- 12:00 推送"拍菜单发我" → 用户发照片
- 12:05 我回复推荐配食（如"✅ 番茄炒蛋+清炒时蔬+半碗饭"）
- 12:30 推送"午间快走20min"

## 常见陷阱

1. ❌ **假设用户居家办公** → ✅ 必须确认工作地点和通勤方式
2. ❌ **删除用户的既有习惯**（午休、协作时间） → ✅ 嵌入而非替换
3. ❌ **忽略配偶的下班时间和交流需求** → ✅ P0非谈判项
4. ❌ **设计"单独运动时间块"** → ✅ 全部嵌入已有时间缝隙
5. ❌ **用理想熄灯时间无视家庭节奏** → ✅ 先问再定
6. ❌ **方案一次性输出，不迭代** → ✅ 分多轮收集约束、逐步逼近可行方案
7. ❌ **把通勤看作纯损耗时间** → ✅ 通勤=阅读/播客时间（地铁），或运动机会（走路/骑行）
8. ❌ **方案设计完就结束** → ✅ 必须搭建执行系统（定时提醒+打卡追踪）
9. ❌ **一周七天同一张时间表** → ✅ 分日型设计（识别学校安排/配偶时间/个人活动的差异）
10. ❌ **用户回复了但没记录** → ✅ 每条回复都要记录到tracker日志
11. ❌ **提醒内容只有干巴巴的指令** → ✅ 每条提醒追加一条相关小知识，增加生活性和用户粘性
