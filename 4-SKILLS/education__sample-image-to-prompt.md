## 二、提示词模板库（直接粘贴使用）

### 2.1 生字表 `shengzibiao`

```text
你是小学语文课本内容提取器，专门提取附录中的生字表/识字表。

【任务边界】
- 只提取课号及其对应的生字
- 提取每个生字的拼音（如果有）
- 忽略：页码、页眉页脚、"生字表""识字表"等标题文字
- 忽略：底部统计文字（如"共250个生字"）

【排版适配规则】
这套模板需要适配以下三种排版，请自动识别并处理：

1. 【两行式】每课两行：第一行拼音，第二行汉字
   → 拼音和汉字按同样顺序一一对应

2. 【单行式】每课一行：课号+汉字，拼音在相邻行
   → 如果相邻行有拼音序列，与汉字一一对应

3. 【识字表式】每课一行：课号+汉字(拼音)
   → 直接提取每个汉字及其括号内的拼音

【关键约束】
- 课号必须准确提取（数字或"语文园地"）
- 每个字单独成条，不要合并
- 如果某课既有拼音又无拼音，用 null 标记缺拼音的字
- 看不清楚的字输出 null，不要猜测

【输出格式】
严格按以下JSON格式输出，不要用```代码块包裹，直接输出纯JSON字符串。
{"type":"shengzibiao","lessons":[{"lesson":25,"chars":[{"char":"缝","pinyin":"féng"},{"char":"箱","pinyin":"xiāng"}]}]}
```

### 2.2 会认字 `cn_recog`

```text
你是小学语文练习册会认字内容提取器。

【任务边界】
- 只提取每行的核心汉字、拼音和组词
- 忽略：题号（A①②③等）、页码、标题
- 忽略：选项标记、打分区域、批改痕迹

【提取规则】
1. 核心汉字 = 每行最左侧的单个汉字
2. 拼音 = 核心汉字对应的完整拼音（注意声调）
3. 组词 = 核心汉字右侧列出的所有组词
4. 一个核心汉字的所有组词合并到一条记录

【输出格式】
{"type":"cn_recog","chars":[{"char":"存","pinyin":"cún","words":["生存","存款","保存"]}]}
```

### 2.3 会写字单字表 `cn_write`

```text
你是小学语文课本会写字内容提取器。

【任务边界】
- 只提取每行列出的单个汉字及其拼音
- 忽略：页码、页眉页脚、标题
- 忽略：行号、序号

【提取规则】
- 如果是表格：每行第一列是汉字，第二列是拼音
- 如果是列表：每个条目包含一个汉字和它的拼音

【输出格式】
{"type":"cn_write","chars":[{"char":"晨","pinyin":"chén"},{"char":"绒","pinyin":"róng"}]}
```

### 2.4 会写字组词拆解表 `cn_write_table`

```text
你是小学语文课本会写字内容提取器。

【任务边界】
- 只提取表格中每行的组词、及其拆解出的每个汉字和拼音
- 忽略：页码、页眉页脚
- 忽略：表格中"标识"列的内容（如A1234567）和"频率"列的数字

【⚠️ 正确理解表格结构——勿与会认字表混淆】
这是一个「会写字组词拆解表」（cn_write_table），其结构与常见的会认字表不同：
- 每行的左侧是一个完整组词（如"生存""回复""官绅"）
- 该行中间的「汉字」列包含该组词拆出的多个单字（如"生 存""回 复""官 绅"）
- 每个单字下方或旁边有其拼音
- 右侧有"标识"和"频率"列，请忽略

【输出格式】
{"type":"cn_write_table","rows":[
  {"word":"生存","chars":[{"char":"生","pinyin":"shēng"},{"char":"存","pinyin":"cún"}]},
  {"word":"回复","chars":[{"char":"回","pinyin":"huí"},{"char":"复","pinyin":"fù"}]}
]}

直接输出纯JSON字符串，不要用代码块包裹。
```

### 2.5 看拼音写词语 `cishu`

```text
你是小学语文试卷/练习册内容提取器，专门提取「看拼音写词语」题型。

【任务边界】
- 只提取每个拼音及其对应的词语
- 保留声调
- 忽略：标题、页号

【输出格式】
{"type":"cishu","entries":[{"word":"乌黑","pinyin":"wūhēi"},{"word":"活泼","pinyin":"huópō"}]}
```

### 2.6 词语表 `cn_words`

```text
你是小学语文课本附录词语表提取器。

【任务边界】
- 提取课号和对应的词语列表
- 词语之间用空格或斜杠分隔
- 一课跨多行时合并到同一课

【排版适配】
- 如果图片有课号：课号分组输出
- 如果图片无课号或课号可以忽略：扁平列表输出

【输出格式 — 课号分组】
{"type":"cn_words","lessons":[{"lesson":2,"words":["乌黑","活泼","春日"]}]}

【输出格式 — 扁平（无课号）】
{"type":"cn_words","words":["乌黑","活泼","春日","轻风","吹拂"]}
```

### 2.7 单字填空练习 `write_char`

```text
你是小学语文练习册单字填空内容提取器。

【任务边界】
- 提取每个填空的拼音提示、提示字（如果有）、正确答案和所在的词语
- 忽略：题号、打分

【输出格式】
{"type":"write_char","pairs":[
  {"pinyin":"bù","hint":"棉","answer":"布","word":"棉布"},
  {"pinyin":"qì","hint":"机","answer":"器","word":"机器"}
]}
```

### 2.8 组词拆解 `word_comp`

```text
你是小学语文组词练习内容提取器。

【任务边界】
- 提取每个核心汉字、其拼音、及其组成的词语

【输出格式】
{"type":"word_comp","entries":[
  {"pinyin":"fēng","word":"丰收","core_char":"丰"},
  {"pinyin":"shōu","word":"丰收","core_char":"收"}
]}
```

## 三、编码时常见决策速查

### 3.1 该用哪个提示词？

收到用户图片 → 先执行 §四的前置流程确认排版 → 根据用户确认的类型，回 §二 取对应提示词。

### 3.2 分词格式问题

OCR 返回的词语表词语可能有不同分隔符：

```python
# 安全分词策略（按优先级）：
# 1. 如果分隔符是斜杠 / → 按 / split
# 2. 如果分隔符是空格 → 按空格 split
# 3. 混合情况 → 先将 / 替换为空格，再统一 split
words_str = re.split(r'[/\s]+', raw_text.strip())
# 过滤：单字、纯数字、空串
words = [w for w in words_str if len(w) >= 2 and not w.isdigit()]
```

### 3.3 大列表输出防截断（≥50条）

```text
不要输出JSON数组，改为纯文本逗号分隔格式：
词语1,词语2,词语3,...
后端收到后再 split(",") 转为列表。
```

### 3.4 JSON安全网

每个 OCR 路由保留 `_extract_json()` 兜底函数（极少情况模型仍会加代码块包裹）：

```python
def _extract_json(text):
    # 移除可能的 ```json 包裹
    text = re.sub(r'^```(?:json)?\s*', '', text.strip())
    text = re.sub(r'\s*```$', '', text)
    return json.loads(text)
```

## 四、前置确认流程（收到样图后执行）

> ⚠️ **不要跳过这一步直接粘贴提示词**。排版判断错了，提示词再精准也没用。

### Step ①：批量观察

用 vision 工具一次性看完所有图片，归纳排版类型：

```text
vision(prompt='提取这张图片中的所有文字，保留原始排版顺序，描述内容分布。')
```

看完后汇总成表格给用户：

```
| 图片 | 排版类型 | 特征 |
|:-----|:---------|:-----|
| 图1 | XX类型 | XX特征 |
| 图2-3 | XX类型 | XX特征 |
```

### Step ②：用户确认

等待用户回复。如果用户说"总体方向没问题"或逐条修正后，再回 §二 取提示词。

## 五、提示词设计原则（新增类型时参考）

当现有8种类型不覆盖新图片时，按以下原则设计新提示词：

**三层结构法**：
```
Layer 1 — 任务边界："你是XX提取器。只提取XX，忽略XX。"
Layer 2 — 输出契约：JSON Schema + 示例
Layer 3 — 质量约束："看不清楚输出null，不要猜测"
```

**关键原则**：
1. 提示词必须基于**实际图片**的排版写，不要抽象描述
2. 同类内容有子格式时，加「排版适配规则」段让模型自动适配
3. 输出格式包含 few-shot 示例（至少1个完整JSON对象）
4. 大列表用紧凑纯文本格式避免截断（见 §三-3）

## 六、陷阱清单（编码阶段容易出问题的点）

| 陷阱 | 表现 | 修正方法 |
|:-----|:-----|:---------|
| **cn_recog vs cn_write_table 混淆** | 两者都含汉字+组词，但数据关系相反 | 看列标题："汉字、组词"→1汉字多词 vs "组词、汉字"→1词拆多字 |
| **词语表分隔符误判** | 只用空格split漏掉斜杠分隔的词 | 用 `/[\s/]+/` 正则split |
| **大列表JSON截断** | 80个词的数组只输出前20个 | 改用紧凑纯文本格式 |
| **跳步直接用提示词** | 没确认排版就粘贴 → 类型不对白干 | 先执行 §四 确认流程 |
