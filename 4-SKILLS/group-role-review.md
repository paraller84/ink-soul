# 群组角色复盘审核机制

## 触发条件
1. **定时触发**：每月1次（通过 cron 任务）
2. **按需触发**：用户主动要求复盘
3. **事件触发**：同一群连续3次以上出现越界提醒移交

## 审核维度

### D1 任务分布分析
- 统计最近周期内各群接收的任务类型分布
- 识别高频任务类型 vs 低频任务类型
- 标记出现但当前角色配置未覆盖的任务类型

### D2 角色覆盖率
- 当前角色配置是否能覆盖前5高频任务类型？
- 是否存在始终闲置、从未激活的角色？
- 是否存在频繁需要但未预设的角色？

### D3 越界移交频率
- 统计各群的"提醒移交"事件次数
- 高频移交的群 → 检查边界定义是否需调整
- 低频/零移交 → 边界定义良好

### D4 常驻角色评估
- 各群的常驻核心角色是否仍为核心？
- 是否有角色应从"按需"升格为"常驻"？
- 是否有常驻角色从未被使用？

## 输出格式

评估结果以结构化报告输出，包含：
- 各群得分雷达（D1-D4 每维度 1-5 分）
- 具体调整建议（优先排序）
- 是否需要更新协议文档

## 更新流程
1. 执行评估 → 2. 输出报告给用户 → 3. 用户决策 → 4. 更新群组角色自报家门协议文档 → 5. 固化调整（更新记忆+技能）
