# 夜雨的工作风格

## 核心原则
- **B/A/A 模式**：①一次性全量实施（不分期）②立即启动（不等待）③直接开干（不额外讨论）
- **只复制不删除**：原文件永不触碰，保留完整历史
- **先梳理清单再动手**：每一个任务先列出完整事项

## 编码要求
- 所有编码必须走 C005 五角色流程：
  需求分析 → 代码学习 → 架构设计 → 工程实现 → 质量测试 → 集成交付
- 多步骤任务（3+步）用 execute_code 批量编排
- Bug 修复先走 C007 Mode D 审核

## 沟通规则
- 每次输出标注模型名称
- 每次任务完成后告知「已完成」
- 领导阅读类文档用条目 + 表格，避免散文

## 质量理念
- prefer fixing pipeline over cleaning up messes
- 全链路追根溯源（从数据来源到最终展示），不只报告 Bug

## 报告/PPT 风格
- 结构：目标 → 推进 → 优化
- 原因分析覆盖所有问题类型
- 技术表述精确
- 竞聘报告必需自我简介关键词页
- 实打实的举措，禁用套话

## 内容创作风格
- 亲身经历 → 实战文（如 Hermes 体系/教育系统）
- 仅观察 → 思考文（行业观察与思考）
- 禁止虚构案例冒充亲身经历
- 区分「实战文」和「思考文」

## 决策风格
- 极其注重细节和架构正确性
- 信任 C005 流程，授权后不过度干预执行细节
- 信任顾问团方案后不要求分期交付
- 视觉偏好：深色科技风主题
- 追求视觉一致性和沉浸式体验

## 邮件风格
- 输出草稿 → 用户微调后回传 → 差异比对学习风格
- 三类风格：
  - 翟总：强势结论型
  - 跨部门：稳妥专业型
  - 同事：指令清晰型

## 教育辅导
- 数学（C003）：试卷分析 → 错题本 → 每日练习 → 复习宝典
- 英语（C008）：语法 SM-2 间隔比词汇更长 (1→3→7→14→30天)
- 语文作文：具体可操作步骤式方法，紧扣教材单元练习
